Schedule Planner

Description

The idea came from the company where I worked. I was a kitchen helper and transporter in the kitchen for the hospitals in Szeged. I was employed through an employment agency and our schedule was made weekly based on the messages in which we wrote when we had free time to work. This program is an exercise to make a software that can help one to manage these schedules better. It was never published and used.

The program is available only in Hungarian.

When you start the program, you first see the main menu, from here can you reach every feature directly.

The first item is for handling the workers (Dolgozók kezelése = Manage workes). Here you have the list of the actual workers. You can add workers, modify their data or delete them. You can save the modifications by clicking Adatok mentése (Save data) button.

The next item is for handling the work schedule requests from the company (Munkarend kezelése = Manage company requests). Here you can work set, how many workers does the company need for a given week for the different shifts. The program later uses this data as a limit to assigning workers in a given shift. You can change the available shifts in the shift manager (Műszakok kezelése = Manage shifts).

The third item is for handling the workers' requests (Ráérések kezelése = Manage worker requests). This item can be reached from the company handling interface. Here you can set, when would a given worker like to work in a given week. You can save the modifications by clicking Ráérést lead (Set request) button.

In the next item you can create the schedule. Here you can see a table that contains the list of workers for every shift of every day of a given week. By setting a tick you assign the worker to the shift. You can create the whole schedule by yourself, create it partly and then let the algorithm to finish it, or let the algoritm do the work completely. You can finalize and save the process by clicking Beosztás készítése (Create schedule) button. The completed schedule can be exported to an Excel file (Export xlxs-be = Export to xlsx).

Next you have the help (Súgó = Help). With the last button you can exit the program (Kilépés = Exit).

Source

Modules used:

  • sqlite3 for database management
  • openpyxl for exporting into Excel

The program's graphical interface is made with tkinter (a version made with PyQt5 also exists). The SHScheduler class is a child of tkinter's Frame class in order to easily place the application onto a tkinter window.

The initialization method loads the database and and creates the main menu window. After that, the code is divided into sections based on the software's working principle (the process of planning schedules), as it is written in the description above. Each section is structured basically the same: first we define methods for creating and managing the graphical user interface, then methods for interacting with the database.

Forráskód

Felhasznált modulok:

  • sqlite3 adatbáziskezeléshez
  • openpyxl Excel-be történő exportáláshoz

A program grafikus felülete tkinter-rel készült (létezik egy PyQt5-tel készült változat is). Az SHScheduler osztály a tkinter Frame osztályából öröklődik azért, hogy az alkalmazást egyszerű legyen elhelyezni egy tkinter ablakban.

Az inicializáló metódus betölti az adatbázist és létrehozza a főmenü ablakot. A kód a program működési elve (a beosztástervezés folyamata) szerint van szakaszokra osztva, ahogy a leírásban olvasható. Minden szakasz hasonlóan épül fel: először a grafikus felületet létrehozó és kezelő metódusok vannak definiálva, aztán az adatbázist kezelőek.
